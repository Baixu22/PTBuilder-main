# Packet Tracer Builder 使用说明

## 一、简介

PTBuilder 是一个 Cisco Packet Tracer 的扩展插件，允许你通过编写 JavaScript 代码来**自动创建网络拓扑**、**连接设备**和**配置设备**。非常适合 AI 辅助生成拓扑或批量创建网络设备。

---

## 二、安装方法

1. 下载 `Builder.pts` 文件
2. 打开 Packet Tracer，点击菜单 **Extensions → Scripting → Configure PT Script Modules**
3. 点击 **Add...** 按钮，选择 `Builder.pts` 文件
4. 安装完成后，通过 **Extensions → Builder Code Editor** 打开代码编辑器

## 三、核心 API 函数

### 1. `addDevice(deviceName, deviceModel, x, y)` — 添加设备

| 参数 | 类型 | 说明 |
|------|------|------|
| `deviceName` | string | 设备名称，如 `"R1"`、`"S1"`、`"PC1"` |
| `deviceModel` | string | 设备型号（见下方设备型号表） |
| `x` | number | 工作区 X 坐标（水平位置） |
| `y` | number | 工作区 Y 坐标（垂直位置） |

```javascript
addDevice("R1", "2911", 100, 100);       // 添加一台路由器
addDevice("S1", "2960-24TT", 200, 100);  // 添加一台交换机
addDevice("PC1", "PC-PT", 300, 100);     // 添加一台 PC
```

### 2. `addLink(device1, interface1, device2, interface2, linkType)` — 连接设备

| 参数 | 类型 | 说明 |
|------|------|------|
| `device1` | string | 设备1名称 |
| `interface1` | string | 设备1的接口名 |
| `device2` | string | 设备2名称 |
| `interface2` | string | 设备2的接口名 |
| `linkType` | string | 线缆类型（见下方线缆类型表） |

```javascript
addLink("R1", "GigabitEthernet0/1", "S1", "GigabitEthernet0/1", "straight");
addLink("S1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");
```

### 3. `addModule(deviceName, slot, model)` — 添加模块

| 参数 | 类型 | 说明 |
|------|------|------|
| `deviceName` | string | 目标设备名称 |
| `slot` | number | 插槽编号（从 1 开始） |
| `model` | string | 模块型号（见下方模块型号表） |

```javascript
addModule("R1", 1, "HWIC-4ESW");    // 给路由器添加4口交换模块
addModule("R1", 2, "NM-2W");        // 给路由器添加无线模块
```

### 4. `configureIosDevice(deviceName, commands)` — 配置 IOS 设备（路由器/交换机/多层交换机）

| 参数 | 类型 | 说明 |
|------|------|------|
| `deviceName` | string | 设备名称 |
| `commands` | string | 多行 IOS 命令字符串（用 `\n` 分隔） |

```javascript
configureIosDevice("R1", `
enable
configure terminal
interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
router rip
version 2
network 192.168.1.0
end
`);
```

### 5. `configurePcIp(deviceName, dhcpEnabled, ip, subnetMask, gateway, dns)` — 配置 PC 的 IP

| 参数 | 类型 | 说明 |
|------|------|------|
| `deviceName` | string | PC 设备名称 |
| `dhcpEnabled` | boolean | 是否启用 DHCP（可选） |
| `ip` | string | IP 地址（可选） |
| `subnetMask` | string | 子网掩码（可选） |
| `gateway` | string | 默认网关（可选） |
| `dns` | string | DNS 服务器（可选） |

```javascript
configurePcIp("PC1", false, "192.168.1.10", "255.255.255.0", "192.168.1.1");
configurePcIp("PC2", true);  // 启用 DHCP
```

### 6. `getDevices(filter, startsWith)` — 获取设备列表

| 参数 | 类型 | 说明 |
|------|------|------|
| `filter` | string/number/array | 按设备类型过滤（可选） |
| `startsWith` | string | 按名称前缀过滤（可选） |

```javascript
var allRouters = getDevices("router");           // 获取所有路由器
var allSwitches = getDevices("switch");          // 获取所有交换机
var sDevices = getDevices(undefined, "S");       // 获取名称以S开头的设备
var mixed = getDevices(["router", "switch"]);    // 获取路由器和交换机
```

## 八、实战拓扑示例

### 示例 1：简单家庭网络（路由器 + 交换机 + 2台PC）

```javascript
// 添加设备
addDevice("R1", "2911", 200, 100);
addDevice("S1", "2960-24TT", 200, 250);
addDevice("PC1", "PC-PT", 100, 400);
addDevice("PC2", "PC-PT", 300, 400);

// 连接线缆
addLink("R1", "GigabitEthernet0/1", "S1", "GigabitEthernet0/1", "straight");
addLink("S1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");
addLink("S1", "FastEthernet0/2", "PC2", "FastEthernet0", "straight");

// 配置路由器接口
configureIosDevice("R1", `
enable
configure terminal
interface GigabitEthernet0/1
ip address 192.168.1.1 255.255.255.0
no shutdown
end
`);

// 配置 PC IP
configurePcIp("PC1", false, "192.168.1.10", "255.255.255.0", "192.168.1.1");
configurePcIp("PC2", false, "192.168.1.20", "255.255.255.0", "192.168.1.1");
```

### 示例 2：路由器互联（WAN 串行链路 + RIP 路由）

```javascript
// 设备
addDevice("R1", "2911", 100, 200);
addDevice("R2", "2911", 500, 200);
addDevice("S1", "2960-24TT", 100, 350);
addDevice("S2", "2960-24TT", 500, 350);
addDevice("PC1", "PC-PT", 50, 450);
addDevice("PC2", "PC-PT", 550, 450);

// 串行链路连接路由器
addLink("R1", "Serial0/0/0", "R2", "Serial0/0/0", "serial");
// 局域网连接
addLink("R1", "GigabitEthernet0/1", "S1", "GigabitEthernet0/1", "straight");
addLink("R2", "GigabitEthernet0/1", "S2", "GigabitEthernet0/1", "straight");
addLink("S1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");
addLink("S2", "FastEthernet0/1", "PC2", "FastEthernet0", "straight");

// 配置 R1
configureIosDevice("R1", `
enable
configure terminal
interface Serial0/0/0
ip address 10.0.0.1 255.255.255.252
clock rate 64000
no shutdown
exit
interface GigabitEthernet0/1
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
router rip
version 2
network 10.0.0.0
network 192.168.1.0
end
`);

// 配置 R2
configureIosDevice("R2", `
enable
configure terminal
interface Serial0/0/0
ip address 10.0.0.2 255.255.255.252
no shutdown
exit
interface GigabitEthernet0/1
ip address 192.168.2.1 255.255.255.0
no shutdown
exit
router rip
version 2
network 10.0.0.0
network 192.168.2.0
end
`);

// 配置 PC
configurePcIp("PC1", false, "192.168.1.10", "255.255.255.0", "192.168.1.1");
configurePcIp("PC2", false, "192.168.2.10", "255.255.255.0", "192.168.2.1");
```

### 示例 3：利用循环批量创建（10台PC连接到一个交换机）

```javascript
addDevice("S1", "2960-24TT", 300, 100);

for (let i = 1; i <= 10; i++) {
    addDevice("PC" + i, "PC-PT", (i - 1) * 60 + 30, 300);
    addLink("S1", "FastEthernet0/" + i, "PC" + i, "FastEthernet0", "straight");
    configurePcIp("PC" + i, false, "192.168.1." + (10 + i), "255.255.255.0", "192.168.1.1");
}
```

### 示例 4：无线拓扑（AP + 笔记本 + 手机）

```javascript
addDevice("R1", "2911", 200, 100);
addDevice("AP1", "AccessPoint-PT-AC", 200, 250);
addDevice("Laptop1", "Laptop-PT", 100, 400);
addDevice("Phone1", "SMARTPHONE-PT", 300, 400);

addLink("R1", "GigabitEthernet0/1", "AP1", "Ethernet0", "straight");

// 无线设备无需物理连线，但需要配置 SSID
configureIosDevice("R1", `
enable
configure terminal
interface GigabitEthernet0/1
ip address 192.168.1.1 255.255.255.0
no shutdown
end
`);
```

### 示例 5：VLAN 配置（交换机划分 VLAN）

```javascript
addDevice("S1", "2960-24TT", 200, 200);
addDevice("PC1", "PC-PT", 100, 350);
addDevice("PC2", "PC-PT", 300, 350);

addLink("S1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");
addLink("S1", "FastEthernet0/2", "PC2", "FastEthernet0", "straight");

// 配置 VLAN
configureIosDevice("S1", `
enable
configure terminal
vlan 10
name Sales
exit
vlan 20
name Engineering
exit
interface FastEthernet0/1
switchport mode access
switchport access vlan 10
exit
interface FastEthernet0/2
switchport mode access
switchport access vlan 20
end
`);

configurePcIp("PC1", false, "192.168.10.10", "255.255.255.0");
configurePcIp("PC2", false, "192.168.20.10", "255.255.255.0");
```

### 示例 6：三层交换机实现 VLAN 间路由

```javascript
addDevice("MLS1", "3560-24PS", 200, 100);
addDevice("PC1", "PC-PT", 100, 250);
addDevice("PC2", "PC-PT", 300, 250);

addLink("MLS1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");
addLink("MLS1", "FastEthernet0/2", "PC2", "FastEthernet0", "straight");

configureIosDevice("MLS1", `
enable
configure terminal
ip routing
vlan 10
name Sales
exit
vlan 20
name Engineering
exit
interface FastEthernet0/1
switchport mode access
switchport access vlan 10
exit
interface FastEthernet0/2
switchport mode access
switchport access vlan 20
exit
interface vlan 10
ip address 192.168.10.1 255.255.255.0
no shutdown
exit
interface vlan 20
ip address 192.168.20.1 255.255.255.0
no shutdown
end
`);

configurePcIp("PC1", false, "192.168.10.10", "255.255.255.0", "192.168.10.1");
configurePcIp("PC2", false, "192.168.20.10", "255.255.255.0", "192.168.20.1");
```

### 示例 7：添加模块扩展接口

```javascript
addDevice("R1", "2911", 200, 100);

// 添加模块（需要先关闭设备电源，addModule 会自动处理）
addModule("R1", 1, "HWIC-4ESW");   // 4口交换模块
addModule("R1", 2, "HWIC-2T");     // 2口串行模块
addModule("R1", 3, "NM-2W");       // 无线模块

addDevice("S1", "2960-24TT", 200, 300);
addDevice("R2", "2911", 500, 100);

addLink("R1", "FastEthernet1/0", "S1", "GigabitEthernet0/1", "straight");
addLink("R1", "Serial0/2/0", "R2", "Serial0/0/0", "serial");
```

## 九、坐标布局参考

工作区坐标系统，建议布局方式：

```
Y=0   ──────────────────────────── (顶部)
       路由器行:     y ≈ 100~150
Y=200  交换机行:     y ≈ 250~300
Y=350  终端设备行:   y ≈ 350~450
Y=500 ──────────────────────────── (底部)
       X: 0 ──── 300 ──── 600 (水平间距建议 100~200)
```

## 十、注意事项

1. **接口名称必须精确匹配**设备型号实际存在的接口，否则会连接失败
2. **串行链路**连接路由器时，DCE 端需设置 `clock rate 64000`
3. **`addModule` 会自动关闭再恢复设备电源**，无需手动处理
4. **`configureIosDevice` 会自动执行 `write memory`** 保存配置
5. **无线设备**（手机、平板等）不需要物理线缆连接到 AP
6. **坐标不能重叠**，设备间至少保持 60~100 像素间距
7. 支持完整的 **JavaScript 语法**：循环、条件判断、变量、函数等

## 十一、快速生成拓扑的思路

当你需要描述一个拓扑时，只需告诉我：

1. **有哪些设备**（名称、类型）
2. **怎么连接**（哪个接口连到哪个接口）
3. **需要什么配置**（IP 地址、路由协议、VLAN 等）

我会自动生成对应的 PTBuilder JavaScript 代码，直接粘贴到 Builder Code Editor 中运行即可。
